The Rust Edition Guide

오류로 승격된 경고

요약

  • bare_trait_objects 및 ellipsis_inclusive_range_patterns 린트를 트리거한 코드는 Rust 2021에서 오류가 발생합니다.

세부 정보

두 개의 기존 린트는 Rust 2021에서 하드 오류가 되지만 이러한 린트는 이전 에디션에서는 경고로 유지됩니다.

bare_trait_objects:

트레이트 객체를 식별하기 위해 dyn 키워드를 사용하는 것은 Rust 2021에서 필수가 됩니다.

예를 들어, &MyTrait에 dyn 키워드를 포함하지 않는 다음 코드는 Rust 2021에서 단순한 린트 대신 오류를 생성합니다.

#![allow(unused)]
fn main() {
pub trait MyTrait {}

pub fn my_function(_trait_object: &MyTrait) { // `&dyn MyTrait`여야 합니다
  unimplemented!()
}
}

ellipsis_inclusive_range_patterns:

포괄적 범위 패턴(즉, 끝 값이 범위에 _포함_되는 범위)에 대한 사용되지 않는 ... 구문은 Rust 2021에서 더 이상 허용되지 않습니다. 표현식과 일치하는 ..=로 대체되었습니다.

예를 들어, 패턴에서 ...를 사용하는 다음 코드는 Rust 2021에서 단순한 린트 대신 오류를 생성합니다.

#![allow(unused)]
fn main() {
pub fn less_or_eq_to_100(n: u8) -> bool {
  matches!(n, 0...100) // `0..=100`이어야 합니다
}
}

마이그레이션

Rust 2015 또는 2018 코드가 bare_trait_objects 또는 ellipsis_inclusive_range_patterns에 대한 경고를 생성하지 않고 #![allow()] 또는 다른 메커니즘을 사용하여 이러한 린트를 허용하지 않은 경우 마이그레이션할 필요가 없습니다.

패턴에서 ...를 사용하거나 트레이트 객체와 함께 dyn을 사용하지 않는 모든 크레이트를 자동으로 마이그레이션하려면 cargo fix --edition을 실행할 수 있습니다.

Cargo 첫 걸음

이 섹션에서는 cargo 명령줄 도구에 대한 간략한 개요를 제공합니다. 새로운 패키지를 생성하고, 패키지 내의 크레이트를 컴파일하고, 결과 프로그램을 실행하는 기능을 보여줍니다.

Cargo로 새 패키지를 시작하려면 cargo new를 사용하세요:

$ cargo new hello_world

Cargo는 기본적으로 바이너리 프로그램을 만들기 위해 --bin을 사용합니다. 라이브러리를 만들려면 대신 --lib를 전달합니다.

Cargo가 우리를 위해 무엇을 생성했는지 확인해 봅시다:

$ cd hello_world
$ tree .
.
├── Cargo.toml
└── src
    └── main.rs

1 directory, 2 files

시작하는 데 필요한 모든 것입니다. 먼저 Cargo.toml을 확인해 봅시다:

[package]
name = "hello_world"
version = "0.1.0"
edition = "2024"

[dependencies]

이것을 매니페스트라고 하며, Cargo가 패키지를 컴파일하는 데 필요한 모든 메타데이터를 포함합니다.

src/main.rs의 내용은 다음과 같습니다:

fn main() {
    println!("Hello, world!");
}

Cargo는 우리를 위해 “hello world” 프로그램을 생성했습니다. 이를 바이너리 크레이트라고도 합니다. 컴파일해 봅시다:

$ cargo build
   Compiling hello_world v0.1.0 (file:///path/to/package/hello_world)

그리고 실행합니다:

$ ./target/debug/hello_world
Hello, world!

또한 cargo run을 사용하여 한 번에 컴파일하고 실행할 수도 있습니다:

$ cargo run
     Fresh hello_world v0.1.0 (file:///path/to/package/hello_world)
   Running `target/hello_world`
Hello, world!

새 패키지 만들기

Cargo로 새 패키지를 시작하려면 cargo new를 사용하세요:

$ cargo new hello_world --bin

바이너리 프로그램을 만들고 있으므로 --bin을 전달합니다. 라이브러리를 만들고 있었다면 --lib를 전달했을 것입니다. 이것은 또한 기본적으로 새로운 git 저장소를 초기화합니다. 원하지 않으면 --vcs none을 전달하세요.

Cargo가 우리를 위해 무엇을 생성했는지 확인해 봅시다:

$ cd hello_world
$ tree .
.
├── Cargo.toml
└── src
    └── main.rs

1 directory, 2 files

Cargo.toml을 자세히 살펴봅시다:

[package]
name = "hello_world"
version = "0.1.0"
edition = "2024"

[dependencies]

이것을 매니페스트라고 하며, Cargo가 패키지를 컴파일하는 데 필요한 모든 메타데이터를 포함합니다. 이 파일은 TOML 형식(/tɑməl/로 발음)으로 작성되었습니다.

src/main.rs의 내용은 다음과 같습니다:

fn main() {
    println!("Hello, world!");
}

Cargo는 “hello world” 프로그램을 생성했습니다. 이를 [바이너리 크레이트](../appendix/glossary.md#crate “크레이트” (용어집 항목)“)라고도 합니다. 컴파일해 봅시다:

$ cargo build
   Compiling hello_world v0.1.0 (file:///path/to/package/hello_world)

그리고 실행합니다:

$ ./target/debug/hello_world
Hello, world!

cargo run을 사용하여 한 번에 컴파일하고 실행할 수도 있습니다 (마지막으로 컴파일한 이후 변경 사항이 없으면 Compiling 줄이 표시되지 않습니다):

$ cargo run
   Compiling hello_world v0.1.0 (file:///path/to/package/hello_world)
     Running `target/debug/hello_world`
Hello, world!

이제 Cargo.lock이라는 새 파일이 표시됩니다. 여기에는 의존성에 대한 정보가 포함되어 있습니다. 아직 의존성이 없으므로 그다지 흥미롭지 않습니다.

릴리스 준비가 되면 cargo build --release를 사용하여 최적화가 켜진 상태로 파일을 컴파일할 수 있습니다:

$ cargo build --release
   Compiling hello_world v0.1.0 (file:///path/to/package/hello_world)

cargo build --release는 결과 바이너리를 target/debug 대신 target/release에 넣습니다.

디버그 모드에서 컴파일하는 것이 개발의 기본값입니다. 컴파일러가 최적화를 수행하지 않기 때문에 컴파일 시간이 더 짧지만 코드는 더 느리게 실행됩니다. 릴리스 모드는 컴파일하는 데 시간이 더 오래 걸리지만 코드는 더 빠르게 실행됩니다.

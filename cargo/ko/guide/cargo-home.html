Cargo 홈

“Cargo 홈”은 다운로드 및 소스 캐시 역할을 합니다. 크레이트를 빌드할 때 Cargo는 다운로드된 빌드 의존성을 Cargo 홈에 저장합니다. CARGO_HOME 환경 변수를 설정하여 Cargo 홈의 위치를 변경할 수 있습니다. home 크레이트는 Rust 크레이트 내에서 이 정보가 필요한 경우 이 위치를 가져오는 API를 제공합니다. 기본적으로 Cargo 홈은 $HOME/.cargo/에 있습니다.

Please note that the internal structure of the Cargo home is not stabilized and may be subject to change at any time.

The Cargo home consists of following components:

파일:

  • config.toml Cargo의 전역 설정 파일입니다. 참조의 config 항목을 확인하세요.

  • credentials.toml Private login credentials from cargo login in order to log in to a registry.

  • .crates.toml, .crates2.json These hidden files contain package information of crates installed via cargo install. Do NOT edit by hand!

디렉토리:

  • bin bin 디렉토리에는 cargo install 또는 rustup를 통해 설치된 크레이트의 실행 파일이 포함되어 있습니다. 이 바이너리들을 접근 가능하게 하려면 디렉토리의 경로를 $PATH 환경 변수에 추가하세요.

  • git Git 소스는 여기에 저장됩니다:

    • git/db When a crate depends on a git repository, Cargo clones the repo as a bare repo into this directory and updates it if necessary.

    • git/checkouts If a git source is used, the required commit of the repo is checked out from the bare repo inside git/db into this directory. This provides the compiler with the actual files contained in the repo of the commit specified for that dependency. Multiple checkouts of different commits of the same repo are possible.

  • registry crates.io와 같은 크레이트 레지스트리의 패키지 및 메타데이터가 여기에 있습니다.

    • registry/index 인덱스는 레지스트리의 모든 사용 가능한 크레이트의 메타데이터(버전, 의존성 등)를 포함하는 bare git 저장소입니다.

    • registry/cache 다운로드된 의존성은 캐시에 저장됩니다. 크레이트는 .crate 확장자로 이름이 지정된 압축된 gzip 아카이브입니다.

    • registry/src 다운로드한 .crate 아카이브가 패키지에 필요한 경우, registry/src 폴더에 압축이 풀리고 rustc가 .rs 파일을 찾을 수 있습니다.

CI에서 Cargo 홈 캐싱하기

지속적인 통합 중에 모든 크레이트 의존성을 다시 다운로드하지 않으려면 $CARGO_HOME 디렉토리를 캐시할 수 있습니다. 그러나 전체 디렉토리를 캐시하는 것은 다운로드된 소스를 두 번 포함하므로 비효율적인 경우가 많습니다. serde 1.0.92와 같은 크레이트에 의존하고 전체 $CARGO_HOME을 캐시하면 실제로는 소스를 두 번 캐시하게 됩니다. registry/cache 내부의 serde-1.0.92.crate와 registry/src 내부의 serde의 압축 해제된 .rs 파일입니다. 캐시를 다운로드, 압축 해제, 재압축 및 CI 서버에 다시 업로드하는 데 시간이 걸릴 수 있으므로 빌드 속도가 불필요하게 느려질 수 있습니다.

cargo install로 설치된 바이너리를 캐시하려면 bin/ 폴더와 .crates.toml 및 .crates2.json 파일을 캐시해야 합니다.

빌드 간에 다음 파일 및 디렉토리를 캐시하는 것으로 충분합니다:

  • .crates.toml
  • .crates2.json
  • bin/
  • registry/index/
  • registry/cache/
  • git/db/

프로젝트의 모든 의존성 벤더링하기

cargo vendor 하위 명령을 참조하세요.

캐시 지우기

이론적으로 캐시의 일부를 언제든지 제거할 수 있으며, Cargo는 크레이트에 소스가 필요한 경우 아카이브를 다시 추출하거나 bare repo를 체크아웃하거나 웹에서 소스를 다시 다운로드하여 소스를 복원하기 위해 최선을 다할 것입니다.

또는 cargo-cache 크레이트는 캐시의 선택된 부분만 지우거나 명령줄에서 구성 요소의 크기를 표시하는 간단한 CLI 도구를 제공합니다.

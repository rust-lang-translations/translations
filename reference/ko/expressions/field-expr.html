[expr.field]

필드 접근 표현식

[expr.field.syntax]

구문
필드표현식 :
   표현식 . 식별자

[expr.field.intro]

_필드 표현식_은 구조체 또는 공용체 필드의 위치로 평가되는 장소 표현식입니다.

[expr.field.mut]

피연산자가 가변이면 필드 표현식도 가변입니다.

[expr.field.form]

필드 표현식의 구문은 _컨테이너 피연산자_라고 하는 표현식, 그 다음 ., 그리고 마지막으로 식별자입니다.

[expr.field.not-method-call]

필드 표현식 뒤에는 괄호로 묶인 쉼표로 구분된 표현식 목록이 올 수 없습니다. 이는 대신 메서드 호출 표현식으로 파싱되기 때문입니다. 즉, 호출 표현식의 함수 피연산자가 될 수 없습니다.

참고: 필드 표현식을 호출 표현식에서 사용하려면 괄호로 묶인 표현식으로 감싸십시오.

#![allow(unused)]
fn main() {
struct HoldsCallable<F: Fn()> { callable: F }
let holds_callable = HoldsCallable { callable: || () };

// 잘못됨: 메서드 "callable" 호출로 파싱됨
// holds_callable.callable();

// 유효함
(holds_callable.callable)();
}

예:

mystruct.myfield;
foo().x;
(Struct {a: 10, b: 20}).a;
(mystruct.function_field)() // 필드 표현식을 포함하는 호출 표현식
[expr.field.autoref-deref]

자동 역참조

컨테이너 피연산자의 타입이 피연산자의 가변성에 따라 Deref 또는 DerefMut를 구현하는 경우, 필드 접근이 가능하도록 필요한 횟수만큼 _자동으로 역참조_됩니다. 이 과정을 줄여서 _자동 역참조(autoderef)_라고도 합니다.

[expr.field.borrow]

차용

구조체의 필드 또는 구조체에 대한 참조는 차용 시 별개의 엔티티로 취급됩니다. 구조체가 Drop을 구현하지 않고 지역 변수에 저장되어 있는 경우, 이는 각 필드에서 이동하는 것에도 적용됩니다. 이는 자동 역참조가 Box 이외의 사용자 정의 타입을 통해 수행되는 경우에는 적용되지 않습니다.

#![allow(unused)]
fn main() {
struct A { f1: String, f2: String, f3: String }
let mut x: A;
x = A {
    f1: "f1".to_string(),
    f2: "f2".to_string(),
    f3: "f3".to_string()
};
let a: &mut String = &mut x.f1; // x.f1 가변적으로 차용됨
let b: &String = &x.f2;         // x.f2 불변적으로 차용됨
let c: &String = &x.f2;         // 다시 차용 가능
let d: String = x.f3;           // x.f3에서 이동
}

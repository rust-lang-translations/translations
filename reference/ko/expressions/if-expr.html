[expr.if]

if 및 if let 표현식

if 표현식

[expr.if.syntax]

구문
If표현식 :
   if 표현식구조체 표현식 제외 블록표현식
   (else ( 블록표현식 | If표현식 | IfLet표현식 ) )?

[expr.if.intro]

if 표현식은 프로그램 제어의 조건부 분기입니다. if 표현식의 구문은 조건 피연산자, 그 뒤에 오는 결과 블록, 임의의 수의 else if 조건 및 블록, 그리고 선택적 후행 else 블록입니다.

[expr.if.condition-bool]

조건 피연산자는 불리언 타입이어야 합니다.

[expr.if.condition-true]

조건 피연산자가 true로 평가되면 결과 블록이 실행되고 후속 else if 또는 else 블록은 건너뜁니다.

[expr.if.else-if]

조건 피연산자가 false로 평가되면 결과 블록은 건너뛰고 후속 else if 조건이 평가됩니다.

[expr.if.else]

모든 if 및 else if 조건이 false로 평가되면 else 블록이 실행됩니다.

[expr.if.result]

if 표현식은 실행된 블록과 동일한 값으로 평가되거나, 어떤 블록도 평가되지 않으면 ()로 평가됩니다.

[expr.if.type]

if 표현식은 모든 상황에서 동일한 타입을 가져야 합니다.

#![allow(unused)]
fn main() {
let x = 3;
if x == 4 {
    println!("x는 4");
} else if x == 3 {
    println!("x는 3");
} else {
    println!("x는 다른 것");
}

let y = if 12 * 15 > 150 {
    "더 큼"
} else {
    "더 작음"
};
assert_eq!(y, "더 큼");
}
[expr.if.let]

if let 표현식

[expr.if.let.syntax]

구문
IfLet표현식 :
   if let 패턴 = 검사대상지연 불리언 연산자 표현식 제외 블록표현식
   (else ( 블록표현식 | If표현식 | IfLet표현식 ) )?

[expr.if.let.intro]

if let 표현식은 의미상 if 표현식과 유사하지만, 조건 피연산자 대신 키워드 let, 그 뒤에 패턴, = 그리고 검사 대상(scrutinee) 피연산자가 옵니다.

[expr.if.let.pattern]

검사 대상의 값이 패턴과 일치하면 해당 블록이 실행됩니다.

[expr.if.let.else]

그렇지 않으면 흐름은 다음 else 블록(있는 경우)으로 진행됩니다.

[expr.if.let.result]

if 표현식과 마찬가지로 if let 표현식은 평가되는 블록에 의해 결정되는 값을 가집니다.

#![allow(unused)]
fn main() {
let dish = ("햄", "달걀");

// 패턴이 반박되므로 이 본문은 건너뜁니다
if let ("베이컨", b) = dish {
    println!("베이컨은 {}와(과) 함께 제공됩니다", b);
} else {
    // 대신 이 블록이 평가됩니다.
    println!("베이컨은 제공되지 않습니다");
}

// 이 본문이 실행됩니다
if let ("햄", b) = dish {
    println!("햄은 {}와(과) 함께 제공됩니다", b);
}

if let _ = 5 {
    println!("반박할 수 없는 패턴은 항상 참입니다");
}
}
[expr.if.let.else-if]

if 및 if let 표현식을 섞어 쓸 수 있습니다:

#![allow(unused)]
fn main() {
let x = Some(3);
let a = if let Some(1) = x {
    1
} else if x == Some(2) {
    2
} else if let Some(y) = x {
    y
} else {
    -1
};
assert_eq!(a, 3);
}
[expr.if.let.desugaring]

if let 표현식은 다음과 같이 match 표현식과 동일합니다:

if let PATS = EXPR {
    /* 본문 */
} else {
    /* else */
}

다음과 동일합니다.

match EXPR {
    PATS => { /* 본문 */ },
    _ => { /* else */ },    // else가 없으면 ()
}
[expr.if.let.or-pattern]

여러 패턴을 | 연산자로 지정할 수 있습니다. 이는 match 표현식의 |와 동일한 의미를 갖습니다:

#![allow(unused)]
fn main() {
enum E {
    X(u8),
    Y(u8),
    Z(u8),
}
let v = E::Y(12);
if let E::X(n) | E::Y(n) = v {
    assert_eq!(n, 12);
}
}
[expr.if.let.lazy-bool]

표현식은 지연 불리언 연산자 표현식이 될 수 없습니다. 지연 불리언 연산자의 사용은 언어의 계획된 기능 변경(if-let 체인의 구현 - eRFC 2947 참조)과 모호합니다. 지연 불리언 연산자 표현식을 사용하려면 다음과 같이 괄호를 사용하여 달성할 수 있습니다:

// 전...
if let PAT = EXPR && EXPR { .. }

// 후...
if let PAT = ( EXPR && EXPR ) { .. }

// 전...
if let PAT = EXPR || EXPR { .. }

// 후...
if let PAT = ( EXPR || EXPR ) { .. }

[expr.loop]

루프 및 기타 중단 가능한 표현식

[expr.loop.syntax]

구문
루프표현식 :
   루프레이블? (
         무한루프표현식
      | 조건루프표현식
      | 조건패턴루프표현식
      | 반복자루프표현식
      | 레이블블록표현식
   )

[expr.loop.intro]

Rust는 다섯 가지 루프 표현식을 지원합니다:

  • loop 표현식은 무한 루프를 나타냅니다.
  • while 표현식은 조건자가 거짓이 될 때까지 반복합니다.
  • while let 표현식은 패턴을 검사합니다.
  • for 표현식은 반복자에서 값을 추출하여 반복자가 빌 때까지 반복합니다.
  • 레이블 블록 표현식은 루프를 정확히 한 번 실행하지만 break로 루프를 일찍 종료할 수 있습니다.
[expr.loop.break-label]

다섯 가지 유형의 루프 모두 break 표현식과 레이블을 지원합니다.

[expr.loop.continue-label]

레이블 블록 표현식을 제외한 모든 루프는 continue 표현식을 지원합니다.

[expr.loop.explicit-result]

loop와 레이블 블록 표현식만이 의미 있는 값으로의 평가를 지원합니다.

[expr.loop.infinite]

무한 루프

[expr.loop.infinite.syntax]

구문
무한루프표현식 :
   loop 블록표현식

[expr.loop.infinite.intro]

loop 표현식은 본문 실행을 지속적으로 반복합니다: loop { println!("I live."); }.

[expr.loop.infinite.diverging]

연관된 break 표현식이 없는 loop 표현식은 발산하며 ! 타입을 가집니다.

[expr.loop.infinite.break]

연관된 break 표현식(들)을 포함하는 loop 표현식은 종료될 수 있으며, break 표현식(들)의 값과 호환되는 타입을 가져야 합니다.

[expr.loop.while]

조건자 루프

[expr.loop.while.syntax]

구문
조건루프표현식 :
   while 표현식구조체 표현식 제외 블록표현식

[expr.loop.while.intro]

while 루프는 불리언 루프 조건 피연산자를 평가하는 것으로 시작합니다.

[expr.loop.while.condition]

루프 조건 피연산자가 true로 평가되면 루프 본문 블록이 실행된 다음 제어가 루프 조건 피연산자로 돌아갑니다. 루프 조건 표현식이 false로 평가되면 while 표현식이 완료됩니다.

예:

#![allow(unused)]
fn main() {
let mut i = 0;

while i < 10 {
    println!("hello");
    i = i + 1;
}
}
[expr.loop.while.let]

조건 패턴 루프

[expr.loop.while.let.syntax]

구문
조건패턴루프표현식 :
   while let 패턴 = 검사대상지연 불리언 연산자 표현식 제외 블록표현식

[expr.loop.while.let.intro]

while let 루프는 의미상 while 루프와 유사하지만, 조건 표현식 대신 키워드 let, 그 뒤에 패턴, =, 검사 대상(scrutinee) 표현식 및 블록 표현식이 옵니다.

[expr.loop.while.let.condition]

검사 대상의 값이 패턴과 일치하면 루프 본문 블록이 실행된 다음 제어가 패턴 매칭 구문으로 돌아갑니다. 그렇지 않으면 while 표현식이 완료됩니다.

#![allow(unused)]
fn main() {
let mut x = vec![1, 2, 3];

while let Some(y) = x.pop() {
    println!("y = {}", y);
}

while let _ = 5 {
    println!("반박할 수 없는 패턴은 항상 참입니다");
    break;
}
}
[expr.loop.while.let.desugar]

while let 루프는 다음과 같이 match 표현식을 포함하는 loop 표현식과 동일합니다.

'label: while let PATS = EXPR {
    /* 루프 본문 */
}

다음과 동일합니다.

'label: loop {
    match EXPR {
        PATS => { /* 루프 본문 */ },
        _ => break,
    }
}
[expr.loop.while.let.or-pattern]

여러 패턴을 | 연산자로 지정할 수 있습니다. 이는 match 표현식의 |와 동일한 의미를 갖습니다:

#![allow(unused)]
fn main() {
let mut vals = vec![2, 3, 1, 2, 2];
while let Some(v @ 1) | Some(v @ 2) = vals.pop() {
    // 2, 2, 그 다음 1을 출력합니다
    println!("{}", v);
}
}
[expr.loop.while.let.lazy-bool]

if let 표현식의 경우와 마찬가지로, 검사 대상은 지연 불리언 연산자 표현식이 될 수 없습니다.

[expr.loop.for]

반복자 루프

[expr.loop.for.syntax]

구문
반복자루프표현식 :
   for 패턴 in 표현식구조체 표현식 제외 블록표현식

[expr.loop.for.intro]

for 표현식은 std::iter::IntoIterator 구현에서 제공하는 요소를 반복하기 위한 구문 구조입니다.

[expr.loop.for.condition]

반복자가 값을 산출하면 해당 값은 반박할 수 없는 패턴과 매칭되고 루프 본문이 실행된 다음 제어가 for 루프의 헤드로 돌아갑니다. 반복자가 비어 있으면 for 표현식이 완료됩니다.

배열의 내용에 대한 for 루프의 예:

#![allow(unused)]
fn main() {
let v = &["사과", "케이크", "커피"];

for text in v {
    println!("저는 {}를 좋아합니다.", text);
}
}

일련의 정수에 대한 for 루프의 예:

#![allow(unused)]
fn main() {
let mut sum = 0;
for n in 1..11 {
    sum += n;
}
assert_eq!(sum, 55);
}
[expr.loop.for.desugar]

for 루프는 다음과 같이 match 표현식을 포함하는 loop 표현식과 동일합니다:

'label: for PATTERN in iter_expr {
    /* 루프 본문 */
}

다음과 동일합니다.

{
    let result = match IntoIterator::into_iter(iter_expr) {
        mut iter => 'label: loop {
            let mut next;
            match Iterator::next(&mut iter) {
                Option::Some(val) => next = val,
                Option::None => break,
            };
            let PATTERN = next;
            let () = { /* 루프 본문 */ };
        },
    };
    result
}
[expr.loop.for.lang-items]

여기서 IntoIterator, Iterator, Option은 현재 스코프에서 해당 이름이 무엇으로 확인되든 상관없이 항상 표준 라이브러리 항목입니다.

변수 이름 next, iter, val은 설명용일 뿐이며, 실제로 사용자가 입력할 수 있는 이름은 아닙니다.

참고: 외부 match는 iter_expr의 임시 값이 루프가 끝나기 전에 드랍되지 않도록 하는 데 사용됩니다. next는 할당되기 전에 선언되는데, 이는 타입이 더 자주 올바르게 추론되도록 하기 때문입니다.

[expr.loop.label]

루프 레이블

[expr.loop.label.syntax]

구문
루프레이블 :
   라이프타임_또는_레이블 :

[expr.loop.label.intro]

루프 표현식은 선택적으로 _레이블_을 가질 수 있습니다. 레이블은 'foo: loop { break 'foo; }, 'bar: while false {}, 'humbug: for _ in 0..0 {}과 같이 루프 표현식 앞에 라이프타임으로 작성됩니다.

[expr.loop.label.control-flow]

레이블이 있는 경우, 이 루프 내에 중첩된 레이블이 있는 break 및 continue 표현식은 이 루프를 빠져나가거나 제어를 루프 헤드로 반환할 수 있습니다. break 표현식 및 continue 표현식을 참조하십시오.

[expr.loop.label.ref]

레이블은 지역 변수의 위생(hygiene) 및 섀도잉 규칙을 따릅니다. 예를 들어, 이 코드는 “outer loop“를 출력합니다:

#![allow(unused)]
fn main() {
'a: loop {
    'a: loop {
        break 'a;
    }
    print!("outer loop");
    break 'a;
}
}

'_는 유효한 루프 레이블이 아닙니다.

[expr.loop.break]

break 표현식

[expr.loop.break.syntax]

구문
중단표현식 :
   break 라이프타임_또는_레이블? 표현식?

[expr.loop.break.intro]

break를 만나면 연관된 루프 본문의 실행이 즉시 종료됩니다. 예를 들면:

#![allow(unused)]
fn main() {
let mut last = 0;
for x in 1..100 {
    if x > 12 {
        break;
    }
    last = x;
}
assert_eq!(last, 12);
}
[expr.loop.break.label]

break 표현식은 일반적으로 break 표현식을 감싸는 가장 안쪽의 loop, for 또는 while 루프와 연관되지만, 레이블을 사용하여 영향을 받는 둘러싸는 루프를 지정할 수 있습니다. 예:

#![allow(unused)]
fn main() {
'outer: loop {
    while true {
        break 'outer;
    }
}
}
[expr.loop.break.value]

break 표현식은 루프 본문에서만 허용되며, break, break 'label 또는 (아래 참조) break EXPR 또는 break 'label EXPR 중 하나의 형식을 가집니다.

[expr.loop.block-labels]

레이블이 붙은 블록 표현식

구문
레이블블록표현식 :
   블록표현식

[expr.loop.block-labels.intro]

레이블 블록 표현식은 블록 내에서 break 표현식을 사용할 수 있다는 점을 제외하고는 블록 표현식과 정확히 같습니다.

[expr.loop.block-labels.break]

루프와 달리, 레이블 블록 표현식 내의 break 표현식은 반드시 레이블을 가져야 합니다(즉, 레이블은 선택 사항이 아닙니다).

[expr.loop.block-labels.label-required]

마찬가지로, 레이블 블록 표현식은 반드시 레이블로 시작해야 합니다.

#![allow(unused)]
fn main() {
fn do_thing() {}
fn condition_not_met() -> bool { true }
fn do_next_thing() {}
fn do_last_thing() {}
let result = 'block: {
    do_thing();
    if condition_not_met() {
        break 'block 1;
    }
    do_next_thing();
    if condition_not_met() {
        break 'block 2;
    }
    do_last_thing();
    3
};
}
[expr.loop.continue]

continue 표현식

[expr.loop.continue.syntax]

구문
계속표현식 :
   continue 라이프타임_또는_레이블?

[expr.loop.continue.intro]

continue를 만나면 연관된 루프 본문의 현재 반복이 즉시 종료되고, 제어가 루프 _헤드_로 반환됩니다.

[expr.loop.continue.while]

while 루프의 경우, 헤드는 루프를 제어하는 조건 표현식입니다.

[expr.loop.continue.for]

for 루프의 경우, 헤드는 루프를 제어하는 호출 표현식입니다.

[expr.loop.continue.label]

break와 마찬가지로, continue는 일반적으로 가장 안쪽의 둘러싸는 루프와 연관되지만, continue 'label을 사용하여 영향을 받는 루프를 지정할 수 있습니다.

[expr.loop.continue.in-loop-only]

continue 표현식은 루프 본문에서만 허용됩니다.

[expr.loop.break-value]

break와 루프 값

[expr.loop.break-value.intro]

When associated with a loop, a break expression may be used to return a value from that loop, via one of the forms break EXPR or break 'label EXPR, where EXPR is an expression whose result is returned from the loop. For example:

#![allow(unused)]
fn main() {
let (mut a, mut b) = (1, 1);
let result = loop {
    if b > 10 {
        break b;
    }
    let c = a + b;
    a = b;
    b = c;
};
// 피보나치 수열에서 10을 넘는 첫 번째 수:
assert_eq!(result, 13);
}
[expr.loop.break-value.loop]

loop에 연관된 break가 있는 경우 발산하는 것으로 간주되지 않으며, loop는 각 break 표현식과 호환되는 타입을 가져야 합니다. 표현식이 없는 break는 표현식 ()가 있는 break와 동일한 것으로 간주됩니다.

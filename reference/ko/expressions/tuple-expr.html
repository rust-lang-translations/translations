[expr.tuple]

튜플 및 튜플 인덱싱 표현식

튜플 표현식

[expr.tuple.syntax]

구문
튜플표현식 :
   ( 튜플요소? )

튜플요소 :
   ( 표현식 , )+ 표현식?

[expr.tuple.result]

_튜플 표현식_은 튜플 값을 생성합니다.

[expr.tuple.intro]

튜플 표현식의 구문은 _튜플 초기화 피연산자_라고 하는 괄호로 묶인 쉼표로 구분된 표현식 목록입니다.

[expr.tuple.unary-tuple-restriction]

단항(1-ary) 튜플 표현식은 괄호로 묶인 표현식과 구분하기 위해 튜플 초기화 피연산자 뒤에 쉼표가 필요합니다.

[expr.tuple.value]

튜플 표현식은 새로 생성된 튜플 타입 값으로 평가되는 값 표현식입니다.

[expr.tuple.type]

튜플 초기화 피연산자의 수는 생성된 튜플의 항수(arity)입니다.

[expr.tuple.unit]

튜플 초기화 피연산자가 없는 튜플 표현식은 유닛 튜플을 생성합니다.

[expr.tuple.fields]

다른 튜플 표현식의 경우, 첫 번째로 작성된 튜플 초기화 피연산자는 필드 0을 초기화하고 후속 피연산자는 다음으로 높은 필드를 초기화합니다. 예를 들어, 튜플 표현식 ('a', 'b', 'c')에서 'a'는 필드 0의 값을 초기화하고, 'b'는 필드 1, 'c'는 필드 2를 초기화합니다.

튜플 표현식과 그 타입의 예:

표현식유형
()() (유닛)
(0.0, 4.5)(f64, f64)
("x".to_string(), )(String, )
("a", 4usize, true)(&'static str, usize, bool)
[expr.tuple-index]

튜플 인덱싱 표현식

[expr.tuple-index.syntax]

구문
튜플인덱싱표현식 :
   표현식 . 튜플_인덱스

[expr.tuple-index.intro]

_튜플 인덱싱 표현식_은 튜플과 튜플 구조체의 필드에 접근합니다.

튜플 인덱스 표현식의 구문은 _튜플 피연산자_라고 하는 표현식, 그 다음 ., 그리고 마지막으로 튜플 인덱스입니다.

[expr.tuple-index.index-syntax]

_튜플 인덱스_의 구문은 선행 0, 밑줄 또는 접미사가 없는 십진 리터럴입니다. 예를 들어 0과 2는 유효한 튜플 인덱스이지만, 01, 0_, 0i32는 그렇지 않습니다.

[expr.tuple-index.required-type]

튜플 피연산자의 타입은 튜플 타입 또는 튜플 구조체여야 합니다.

[expr.tuple-index.index-name-operand]

튜플 인덱스는 튜플 피연산자 타입의 필드 이름이어야 합니다.

[expr.tuple-index.result]

튜플 인덱스 표현식의 평가는 튜플 피연산자의 평가 외에 부작용이 없습니다. 장소 표현식으로서, 튜플 인덱스와 이름이 같은 튜플 피연산자 필드의 위치로 평가됩니다.

튜플 인덱싱 표현식의 예:

#![allow(unused)]
fn main() {
// 튜플 인덱싱
let pair = ("문자열", 2);
assert_eq!(pair.1, 2);

// 튜플 구조체 인덱싱
struct Point(f32, f32);
let point = Point(1.0, 0.0);
assert_eq!(point.0, 1.0);
assert_eq!(point.1, 0.0);
}

참고: 필드 접근 표현식과 달리, 튜플 인덱스 표현식은 메서드 이름이 숫자가 될 수 없으므로 메서드 호출과 혼동될 수 없기 때문에 호출 표현식의 함수 피연산자가 될 수 있습니다.

참고: 배열과 슬라이스에도 요소가 있지만, 해당 요소에 접근하려면 배열 또는 슬라이스 인덱싱 표현식이나 슬라이스 패턴을 사용해야 합니다.

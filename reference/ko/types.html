The Rust Reference

[type]

타입

[type.intro]

러스트 프로그램의 모든 변수, 아이템, 값은 타입을 가집니다. _값_의 _타입_은 해당 값을 보유한 메모리의 해석 방식과 해당 값에 대해 수행할 수 있는 연산을 정의합니다.

[type.builtin]

내장 타입(Built-in types)은 언어에 긴밀하게 통합되어 있으며, 사용자 정의 타입으로는 흉내 낼 수 없는 복잡한 방식으로 작동합니다.

[type.user-defined]

사용자 정의 타입은 제한된 능력을 갖습니다.

[type.kinds]

타입 목록은 다음과 같습니다:

  • 기본 타입(Primitive types):
    • 불리언 — bool
    • 숫자 — 정수 및 부동 소수점
    • 텍스트 — char 및 str
    • 네버 — ! — 값이 없는 타입
  • 시퀀스 타입:
    • 튜플
    • 배열
    • 슬라이스
  • 사용자 정의 타입:
    • 구조체
    • 열거형
    • 공용체
  • 함수 타입:
    • 함수
    • 클로저
  • 포인터 타입:
    • 참조
    • 원시 포인터
    • 함수 포인터
  • 트레잇 타입:
    • 트레잇 객체
    • Impl 트레잇

[type.name.intro]

위의 Type 구문 규칙에 정의된 _타입 표현식_은 타입을 참조하기 위한 구문입니다. 이는 다음을 참조할 수 있습니다:

[type.name.sequence]
  • 시퀀스 타입 (튜플, 배열, 슬라이스).
[type.name.path]
  • 다음을 참조할 수 있는 타입 경로:
    • 기본 타입 (불리언, 숫자, 텍스트).
    • 아이템으로의 경로 (구조체, 열거형, 공용체, 타입 별칭, 트레잇).
    • Self가 구현 타입인 Self 경로.
    • 제네릭 타입 파라미터.
[type.name.pointer]
  • 포인터 타입 (참조, 원시 포인터, 함수 포인터).
[type.name.inference]
  • 컴파일러에게 타입을 결정하도록 요청하는 추론된 타입.
[type.name.grouped]
  • 모호성을 제거하기 위해 사용되는 괄호.
[type.name.trait]
  • 트레잇 타입: 트레잇 객체 및 impl 트레잇.
[type.name.never]
  • 네버 타입.
[type.name.macro-expansion]
  • 타입 표현식으로 확장되는 매크로.
[type.name.parenthesized]

괄호로 둘러싸인 타입

[type.name.parenthesized.syntax]

괄호로둘러싸인타입 :
   ( 타입 )

[type.name.parenthesized.intro]

어떤 상황에서는 타입의 조합이 모호할 수 있습니다. 모호성을 피하기 위해 타입 주위에 괄호를 사용하십시오. 예를 들어, 참조 타입 내의 타입 바운드를 위한 + 연산자는 바운드가 어디에 적용되는지 불분명하므로 괄호를 사용해야 합니다. 이러한 모호성 제거가 필요한 구문 규칙은 Type 규칙 대신 TypeNoBounds 규칙을 사용합니다.

#![allow(unused)]
fn main() {
use std::any::Any;
type T<'a> = &'a (dyn Any + Send);
}
[type.recursive]

재귀적 타입

[type.recursive.intro]

명목적 타입(Nominal types)인 구조체, 열거형, 공용체는 재귀적일 수 있습니다. 즉, 각 enum 변형이나 struct 또는 union 필드는 자신을 둘러싼 enum 또는 struct 타입 자체를 직접적 또는 간접적으로 참조할 수 있습니다.

[type.recursive.constraint]

이러한 재귀에는 제한 사항이 있습니다:

  • 재귀적 타입은 재귀 과정에 반드시 명목적 타입을 포함해야 합니다(단순한 타입 별칭이나 배열, 튜플과 같은 다른 구조적 타입만으로는 안 됩니다). 따라서 type Rec = &'static [Rec]은 허용되지 않습니다.
  • 재귀적 타입의 크기는 유한해야 합니다. 다시 말해, 해당 타입의 재귀적 필드는 반드시 포인터 타입이어야 합니다.

재귀적 타입과 그 사용 예시:

#![allow(unused)]
fn main() {
enum List<T> {
    Nil,
    Cons(T, Box<List<T>>)
}

let a: List<i32> = List::Cons(7, Box::new(List::Cons(13, Box::new(List::Nil))));
}

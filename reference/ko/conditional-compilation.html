The Rust Reference

[cfg]

조건부 컴파일

[cfg.syntax]

Syntax
ConfigurationPredicate :
      ConfigurationOption
   | ConfigurationAll
   | ConfigurationAny
   | ConfigurationNot

ConfigurationOption :
   IDENTIFIER (= (STRING_LITERAL | RAW_STRING_LITERAL))?

ConfigurationAll
   all ( ConfigurationPredicateList? )

ConfigurationAny
   any ( ConfigurationPredicateList? )

ConfigurationNot
   not ( ConfigurationPredicate )

ConfigurationPredicateList
   ConfigurationPredicate (, ConfigurationPredicate)* ,?

[cfg.general]

_조건부로 컴파일된 소스 코드_는 특정 조건에서만 컴파일되는 소스 코드입니다.

[cfg.attributes-macro]

소스 코드는 cfg 및 cfg_attr 속성과 내장된 cfg 매크로를 사용하여 조건부로 컴파일할 수 있습니다.

[cfg.conditional]

컴파일 여부는 컴파일된 크레이트의 대상 아키텍처, 컴파일러에 전달된 임의의 값 및 아래에 자세히 설명된 기타 사항에 따라 달라질 수 있습니다.

[cfg.predicate]

각 형태의 조건부 컴파일은 참 또는 거짓으로 평가되는 _구성 술어_를 사용합니다. 술어는 다음 중 하나입니다.

[cfg.predicate.option]
  • 구성 옵션. 옵션이 설정되면 술어는 참이고 설정되지 않으면 거짓입니다.
[cfg.predicate.all]
  • all()은 쉼표로 구분된 구성 술어 목록과 함께 사용됩니다. 주어진 모든 술어가 참이거나 목록이 비어 있으면 참입니다.
[cfg.predicate.any]
  • any()은 쉼표로 구분된 구성 술어 목록과 함께 사용됩니다. 주어진 술어 중 하나 이상이 참이면 참입니다. 술어가 없으면 거짓입니다.
[cfg.predicate.not]
  • not()은 구성 술어와 함께 사용됩니다. 술어가 거짓이면 참이고 술어가 참이면 거짓입니다.
[cfg.option-spec]

_구성 옵션_은 이름 또는 키-값 쌍이며 설정되거나 설정되지 않습니다.

[cfg.option-name]

이름은 unix와 같이 단일 식별자로 작성됩니다.

[cfg.option-key-value]

키-값 쌍은 식별자, =, 그리고 문자열로 작성됩니다(예: target_arch = "x86_64").

참고: = 주변의 공백은 무시되므로 foo="bar"와 foo = "bar"는 동일합니다.

[cfg.option-key-uniqueness]

키는 고유할 필요가 없습니다. 예를 들어, feature = "std"와 feature = "serde"를 동시에 설정할 수 있습니다.

[cfg.options.set]

구성 옵션 설정

[cfg.options.general]

어떤 구성 옵션이 설정되는지는 크레이트 컴파일 중에 정적으로 결정됩니다.

[cfg.options.target]

일부 옵션은 컴파일에 대한 데이터를 기반으로 _컴파일러 설정_됩니다.

[cfg.options.other]

다른 옵션은 코드 외부에서 컴파일러에 전달된 입력을 기반으로 _임의로 설정_됩니다.

[cfg.options.crate]

컴파일 중인 크레이트의 소스 코드 내에서 구성 옵션을 설정할 수 없습니다.

참고: rustc의 경우, 임의로 설정된 구성 옵션은 --cfg 플래그를 사용하여 설정됩니다. 지정된 대상에 대한 구성 값은 rustc --print cfg --target $TARGET으로 표시할 수 있습니다.

참고: feature 키가 있는 구성 옵션은 컴파일 시간 옵션 및 선택적 종속성을 지정하기 위해 Cargo에서 사용하는 규칙입니다.

경고: 임의로 설정된 구성 옵션은 컴파일러 설정 구성 옵션과 충돌할 수 있습니다. 예를 들어, Windows 대상으로 컴파일하는 동안 rustc --cfg "unix" program.rs를 수행하고 unix와 windows 구성 옵션을 동시에 설정할 수 있습니다. 이렇게 하는 것은 현명하지 않습니다.

[cfg.target_arch]

target_arch

[cfg.target_arch.gen]

키-값 옵션은 대상의 CPU 아키텍처로 한 번 설정됩니다. 값은 플랫폼의 대상 트리플의 첫 번째 요소와 유사하지만 동일하지는 않습니다.

[cfg.target_arch.values]

예제 값:

  • "x86"
  • "x86_64"
  • "mips"
  • "powerpc"
  • "powerpc64"
  • "arm"
  • "aarch64"
[cfg.target_feature]

target_feature

[cfg.target_feature.general]

현재 컴파일 대상에 사용할 수 있는 각 플랫폼 기능에 대해 설정된 키-값 옵션입니다.

[cfg.target_feature.values]

예제 값:

  • "avx"
  • "avx2"
  • "crt-static"
  • "rdrand"
  • "sse"
  • "sse2"
  • "sse4.1"

사용 가능한 기능에 대한 자세한 내용은 target_feature 속성을 참조하십시오.

[cfg.target_feature.crt_static]

An additional feature of crt-static is available to the target_feature option to indicate that a static C runtime is available.

[cfg.target_os]

target_os

[cfg.target_os.general]

키-값 옵션은 대상의 운영 체제로 한 번 설정됩니다. 이 값은 플랫폼의 대상 트리플의 두 번째 및 세 번째 요소와 유사합니다.

[cfg.target_os.values]

예제 값:

  • "windows"
  • "macos"
  • "ios"
  • "linux"
  • "android"
  • "freebsd"
  • "dragonfly"
  • "openbsd"
  • "netbsd"
  • "none" (임베디드 대상의 경우 일반적)
[cfg.target_family]

target_family

[cfg.target_family.general]

키-값 옵션은 대상이 일반적으로 속하는 운영 체제 또는 아키텍처 제품군과 같이 대상에 대한 보다 일반적인 설명을 제공합니다. target_family 키-값 쌍은 여러 개 설정할 수 있습니다.

[cfg.target_family.values]

예제 값:

  • "unix"
  • "windows"
  • "wasm"
  • "unix"와 "wasm" 모두
[cfg.target_family.unix]

unix 및 windows

unix는 target_family = "unix"가 설정되면 설정됩니다.

[cfg.target_family.windows]

windows는 target_family = "windows"가 설정되면 설정됩니다.

[cfg.target_env]

target_env

[cfg.target_env.general]

사용된 ABI 또는 libc에 대한 정보와 함께 대상 플랫폼에 대한 추가 명확화 정보로 설정된 키-값 옵션입니다. 역사적인 이유로 이 값은 실제로 명확화가 필요할 때만 빈 문자열이 아닌 것으로 정의됩니다. 따라서 예를 들어 많은 GNU 플랫폼에서 이 값은 비어 있습니다. 이 값은 플랫폼의 대상 트리플의 네 번째 요소와 유사합니다. 한 가지 차이점은 gnueabihf와 같은 임베디드 ABI는 단순히 target_env를 "gnu"로 정의한다는 것입니다.

[cfg.target_env.values]

예제 값:

  • ""
  • "gnu"
  • "msvc"
  • "musl"
  • "sgx"
[cfg.target_abi]

target_abi

[cfg.target_abi.general]

대상 ABI에 대한 정보로 target_env를 추가로 명확히 하기 위해 설정된 키-값 옵션입니다.

[cfg.target_abi.disambiguation]

역사적인 이유로 이 값은 실제로 명확화가 필요할 때만 빈 문자열이 아닌 것으로 정의됩니다. 따라서 예를 들어 많은 GNU 플랫폼에서 이 값은 비어 있습니다.

[cfg.target_abi.values]

예제 값:

  • ""
  • "llvm"
  • "eabihf"
  • "abi64"
  • "sim"
  • "macabi"
[cfg.target_endian]

target_endian

키-값 옵션은 대상의 CPU 엔디안에 따라 “little” 또는 “big” 값으로 한 번 설정됩니다.

[cfg.target_pointer_width]

target_pointer_width

[cfg.target_pointer_width.general]

키-값 옵션은 대상의 포인터 너비를 비트 단위로 한 번 설정합니다.

[cfg.target_pointer_width.values]

예제 값:

  • "16"
  • "32"
  • "64"
[cfg.target_vendor]

target_vendor

[cfg.target_vendor.general]

키-값 옵션은 대상의 공급업체로 한 번 설정됩니다.

[cfg.target_vendor.values]

예제 값:

  • "apple"
  • "fortanix"
  • "pc"
  • "unknown"
[cfg.target_has_atomic]

target_has_atomic

[cfg.target_has_atomic.general]

대상이 원자적 로드, 저장 및 비교-스왑 작업을 지원하는 각 비트 너비에 대해 설정된 키-값 옵션입니다.

[cfg.target_has_atomic.stdlib]

이 cfg가 있으면 관련 원자 너비에 대해 안정적인 core::sync::atomic API를 모두 사용할 수 있습니다.

[cfg.target_has_atomic.values]

가능한 값:

  • "8"
  • "16"
  • "32"
  • "64"
  • "128"
  • "ptr"
[cfg.test]

test

테스트 하네스를 컴파일할 때 활성화됩니다. rustc로 --test 플래그를 사용하여 수행됩니다. 테스트 지원에 대한 자세한 내용은 테스팅을 참조하십시오.

[cfg.debug_assertions]

debug_assertions

최적화 없이 컴파일할 때 기본적으로 활성화됩니다. 이는 프로덕션이 아닌 개발 환경에서 추가 디버깅 코드를 활성화하는 데 사용할 수 있습니다. 예를 들어, 표준 라이브러리의 debug_assert! 매크로의 동작을 제어합니다.

[cfg.proc_macro]

proc_macro

컴파일 중인 크레이트가 proc_macro 크레이트 타입으로 컴파일될 때 설정됩니다.

[cfg.panic]

panic

[cfg.panic.general]

패닉 전략에 따라 설정되는 키-값 옵션입니다. 미래에 더 많은 값이 추가될 수 있음에 유의하세요.

[cfg.panic.values]

예제 값:

  • "abort"
  • "unwind"

조건부 컴파일의 형태

[cfg.attr]

cfg 속성

[cfg.attr.syntax]

구문
CfgAttr속성 :
   cfg ( 구성조건자 )

[cfg.attr.general]

cfg 속성은 구성 조건자에 따라 부착된 대상을 조건부로 포함합니다.

[cfg.attr.syntax-explanation]

cfg, (, 구성 조건자, 그리고 마지막으로 )로 작성됩니다.

[cfg.attr.effect]

조건자가 참이면, 대상은 cfg 속성이 없는 상태로 다시 작성됩니다. 조건자가 거짓이면, 대상은 소스 코드에서 제거됩니다.

[cfg.attr.crate-level-attrs]

크레이트 수준의 cfg가 거짓 술어를 가질 때, 동작이 약간 다릅니다. cfg 앞에 오는 모든 크레이트 속성은 유지되고, cfg 뒤에 오는 모든 크레이트 속성은 제거됩니다. 이를 통해 #![cfg(...)]가 크레이트 전체를 제거하더라도 #![no_std]및#![no_core]크레이트가std/core`를 링크하지 않도록 할 수 있습니다.

함수에 대한 몇 가지 예시:

#![allow(unused)]
fn main() {
// 이 함수는 macOS용으로 컴파일할 때만 빌드에 포함됩니다.
#[cfg(target_os = "macos")]
fn macos_only() {
  // ...
}

// 이 함수는 foo 또는 bar 중 하나가 정의된 경우에만 포함됩니다.
#[cfg(any(foo, bar))]
fn needs_foo_or_bar() {
  // ...
}

// 이 함수는 32비트 아키텍처의 unix 계열 OS용으로 컴파일할 때만 포함됩니다.
#[cfg(all(unix, target_pointer_width = "32"))]
fn on_32bit_unix() {
  // ...
}

// 이 함수는 foo가 정의되지 않은 경우에만 포함됩니다.
#[cfg(not(foo))]
fn needs_not_foo() {
  // ...
}

// 이 함수는 패닉 전략이 unwind로 설정된 경우에만 포함됩니다.
#[cfg(panic = "unwind")]
fn when_unwinding() {
  // ...
}

}
[cfg.attr.restriction]

cfg 속성은 속성이 허용되는 모든 곳에서 허용됩니다.

[cfg.cfg_attr]

cfg_attr 속성

[cfg.cfg_attr.syntax]

구문
CfgAttr속성 :
   cfg_attr ( 구성술어 , Cfg속성들? )

Cfg속성들 :
   속성 (, 속성)* ,?

[cfg.cfg_attr.general]

cfg_attr 속성은 구성 술어에 기반하여 조건부로 속성들을 포함합니다.

[cfg.cfg_attr.behaviour]

구성 술어가 참일 때, 이 속성은 술어 뒤에 나열된 속성들로 확장됩니다. 예를 들어, 다음 모듈은 대상에 따라 linux.rs 또는 windows.rs에서 발견됩니다.

#[cfg_attr(target_os = "linux", path = "linux.rs")]
#[cfg_attr(windows, path = "windows.rs")]
mod os;
[cfg.cfg_attr.attribute-list]

0개, 1개 또는 그 이상의 속성이 나열될 수 있습니다. 여러 속성은 각각 별도의 속성으로 확장됩니다. 예를 들어:

#[cfg_attr(feature = "magic", sparkles, crackles)]
fn bewitched() {}

// `magic` 피처 플래그가 활성화되면, 위 코드는 다음과 같이 확장됩니다:
#[sparkles]
#[crackles]
fn bewitched() {}

참고: cfg_attr은 다른 cfg_attr로 확장될 수 있습니다. 예를 들어, #[cfg_attr(target_os = "linux", cfg_attr(feature = "multithreaded", some_other_attribute))]는 유효합니다. 이 예제는 #[cfg_attr(all(target_os = "linux", feature ="multithreaded"), some_other_attribute)]와 동일합니다.

[cfg.cfg_attr.restriction]

cfg_attr 속성은 속성이 허용되는 모든 곳에 허용됩니다.

crate_type 및 crate_name 속성은 cfg_attr과 함께 사용할 수 없습니다.

[cfg.macro]

cfg 매크로

내장된 cfg 매크로는 단일 구성 조건자를 취하며, 조건자가 참이면 true 리터럴로, 거짓이면 false 리터럴로 평가됩니다.

예:

#![allow(unused)]
fn main() {
let machine_kind = if cfg!(unix) {
  "unix"
} else if cfg!(windows) {
  "windows"
} else {
  "unknown"
};

println!("저는 {} 머신에서 실행 중입니다!", machine_kind);
}

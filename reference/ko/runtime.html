The Rust Reference

[runtime]

러스트 런타임

이 섹션은 러스트 런타임의 일부 측면을 정의하는 기능들을 설명합니다.

[runtime.panic_handler]

panic_handler 속성

[runtime.panic_handler.allowed-positions]

_panic_handler 속성_은 오직 시그니처가 fn(&PanicInfo) -> !인 함수에만 적용될 수 있습니다.

[runtime.panic_handler.intro]

이 속성이 표시된 함수는 패닉(panic) 시의 동작을 정의합니다.

[runtime.panic_handler.panic-info]

PanicInfo 구조체는 패닉이 발생한 위치에 대한 정보를 포함합니다.

[runtime.panic_handler.unique]

binary, dylib, 또는 cdylib 크레이트의 의존성 그래프에는 단 하나의 panic_handler 함수가 존재해야 합니다.

아래는 패닉 메시지를 기록한 다음 스레드를 중단시키는 panic_handler 함수의 예시입니다.

#![no_std]

use core::fmt::{self, Write};
use core::panic::PanicInfo;

struct Sink {
    // ..
   _0: (),
}

impl Sink {
    fn new() -> Sink { Sink { _0: () }}
}

impl fmt::Write for Sink {
    fn write_str(&mut self, _: &str) -> fmt::Result { Ok(()) }
}

#[panic_handler]
fn panic(info: &PanicInfo) -> ! {
    let mut sink = Sink::new();

    // "panicked at '$reason', src/main.rs:27:4"를 어떤 `sink`에 기록함
    let _ = writeln!(sink, "{}", info);

    loop {}
}
[runtime.panic_handler.std]

표준 동작

표준 라이브러리는 기본적으로 스택을 언와인딩(unwinding)하는 panic_handler 구현을 제공하지만, 이를 프로세스를 중단(abort)하도록 변경할 수 있습니다. 표준 라이브러리의 패닉 동작은 런타임에 set_hook 함수를 사용하여 수정될 수 있습니다.

[runtime.global_allocator]

global_allocator 속성

_global_allocator 속성_은 GlobalAlloc 트레잇을 구현하는 정적 아이템(static item)에 사용되어 전역 할당자(global allocator)를 설정합니다.

[runtime.windows_subsystem]

windows_subsystem 속성

[runtime.windows_subsystem.intro]

_windows_subsystem 속성_은 윈도우 타겟에서 링크할 때 서브시스템(subsystem)을 설정하기 위해 크레이트 레벨에서 적용될 수 있습니다.

[runtime.windows_subsystem.syntax]

이 속성은 MetaNameValueStr 구문을 사용하여 console 또는 windows 중 하나의 값으로 서브시스템을 지정합니다.

[runtime.windows_subsystem.ignored]

이 속성은 윈도우가 아닌 타겟이나 bin이 아닌 크레이트 타입에서는 무시됩니다.

[runtime.windows_subsystem.console]

“console” 서브시스템이 기본값입니다. 만약 콘솔 프로세스가 기존 콘솔에서 실행된다면 해당 콘솔에 부착되고, 그렇지 않으면 새로운 콘솔 창이 생성됩니다.

[runtime.windows_subsystem.windows]

“windows” 서브시스템은 시작 시 콘솔 창을 표시하고 싶지 않은 GUI 애플리케이션에서 주로 사용됩니다. 이는 기존의 어떤 콘솔과도 분리되어 실행됩니다.

#![allow(unused)]
#![windows_subsystem = "windows"]
fn main() {
}

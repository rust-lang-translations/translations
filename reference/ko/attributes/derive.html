[attributes.derive]

파생

[attributes.derive.intro]

_derive 속성_은 데이터 구조에 대해 새로운 아이템들이 자동으로 생성되도록 합니다.

[attributes.derive.syntax]

이 속성은 구현할 트레잇 목록이나 처리할 derive 매크로 경로를 지정하기 위해 MetaListPaths 구문을 사용합니다.

예를 들어, 다음 코드는 Foo에 대해 PartialEq 및 Clone 트레잇을 위한 impl 아이템을 생성하며, 타입 매개변수 T에는 적절한 impl을 위한 PartialEq 또는 Clone 제약 조건이 부여됩니다.

#![allow(unused)]
fn main() {
#[derive(PartialEq, Clone)]
struct Foo<T> {
    a: i32,
    b: T,
}
}

PartialEq를 위해 생성된 impl은 다음과 동일합니다.

#![allow(unused)]
fn main() {
struct Foo<T> { a: i32, b: T }
impl<T: PartialEq> PartialEq for Foo<T> {
    fn eq(&self, other: &Foo<T>) -> bool {
        self.a == other.a && self.b == other.b
    }
}
}
[attributes.derive.proc-macro]

절차적 매크로를 통해 여러분의 트레잇에 대해서도 derive를 구현할 수 있습니다.

[attributes.derive.automatically_derived]

automatically_derived 속성

_automatically_derived 속성_은 내장 트레잇에 대해 derive 속성에 의해 생성된 구현들에 자동으로 추가됩니다. 이 속성은 직접적인 효과는 없지만, 도구나 진단 린트에서 이렇게 자동으로 생성된 구현들을 감지하는 데 사용될 수 있습니다.

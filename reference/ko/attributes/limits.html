[attributes.limits]

제한

다음 속성들은 컴파일 타임 제한에 영향을 줍니다.

[attributes.limits.recursion_limit]

recursion_limit 속성

[attributes.limits.recursion_limit.intro]

_recursion_limit 속성_은 매크로 확장이나 자동 역참조(auto-dereference)와 같이 잠재적으로 무한히 재귀적인 컴파일 타임 연산의 최대 깊이를 설정하기 위해 크레이트 레벨에 적용될 수 있습니다.

[attributes.limits.recursion_limit.syntax]

이 속성은 재귀 깊이를 지정하기 위해 MetaNameValueStr 구문을 사용합니다.

참고: rustc의 기본값은 128입니다.

#![allow(unused)]
#![recursion_limit = "4"]

fn main() {
macro_rules! a {
    () => { a!(1); };
    (1) => { a!(2); };
    (2) => { a!(3); };
    (3) => { a!(4); };
    (4) => { };
}

// 이는 4보다 큰 재귀 깊이를 요구하므로 확장에 실패합니다.
a!{}
}
#![allow(unused)]
#![recursion_limit = "1"]

fn main() {
// 이는 자동 역참조를 위해 두 번의 재귀 단계가 필요하므로 실패합니다.
(|_: &u8| {})(&&&1);
}
[attributes.limits.type_length_limit]

type_length_limit 속성

참고: 이 제한은 나이틀리(nightly) -Zenforce-type-length-limit 플래그가 활성화된 경우에만 적용됩니다.

자세한 내용은 https://github.com/rust-lang/rust/pull/127670을 참조하세요.

[attributes.limits.type_length_limit.intro]

_type_length_limit 속성_은 모노모르포화 과정에서 구체적인 타입을 생성할 때 수행되는 타입 치환(type substitutions)의 최대 횟수를 제한합니다.

[attributes.limits.type_length_limit.syntax]

이 속성은 크레이트 레벨에 적용되며, 타입 치환 횟수에 기반하여 제한을 설정하기 위해 MetaNameValueStr 구문을 사용합니다.

참고: rustc의 기본값은 1048576입니다.

#![type_length_limit = "4"]

fn f<T>(x: T) {}

// 이는 `f::<((((i32,), i32), i32), i32)>`로 모노모르포화하는 데 4개 이상의 타입
// 요소가 필요하므로 컴파일에 실패합니다.
f(((((1,), 2), 3), 4));

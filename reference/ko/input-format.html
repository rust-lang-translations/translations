The Rust Reference

[input]

입력 형식

[input.intro]

이 장에서는 소스 파일이 토큰 시퀀스로 해석되는 방법을 설명합니다.

프로그램이 파일로 구성되는 방법에 대한 설명은 크레이트 및 소스 파일을 참조하십시오.

[input.encoding]

소스 인코딩

[input.encoding.utf8]

각 소스 파일은 UTF-8로 인코딩된 유니코드 문자 시퀀스로 해석됩니다.

[input.encoding.invalid]

파일이 유효한 UTF-8이 아니면 오류입니다.

[input.byte-order-mark]

바이트 순서 마크 제거

시퀀스의 첫 번째 문자가 U+FEFF(바이트 순서 마크)이면 제거됩니다.

[input.crlf]

CRLF 정규화

U+000D (CR) 문자와 그 뒤에 바로 오는 U+000A (LF) 문자의 각 쌍은 단일 U+000A (LF)로 대체됩니다.

U+000D (CR) 문자의 다른 발생은 그대로 유지됩니다(이들은 공백으로 처리됩니다).

[input.shebang]

쉬뱅 제거

[input.shebang.intro]

남은 시퀀스가 #! 문자로 시작하면, 첫 번째 U+000A (LF)까지의 문자가 시퀀스에서 제거됩니다.

예를 들어, 다음 파일의 첫 번째 줄은 무시됩니다:

#!/usr/bin/env rustx

fn main() {
    println!("안녕하세요!");
}
[input.shebang.inner-attribute]

예외적으로, #! 문자 뒤에 (중간에 있는 주석이나 공백을 무시하고) [ 토큰이 오면 아무것도 제거되지 않습니다. 이는 소스 파일 시작 부분에 있는 내부 속성이 제거되는 것을 방지합니다.

참고: 표준 라이브러리 include! 매크로는 읽는 파일에 바이트 순서 마크 제거, CRLF 정규화 및 쉬뱅 제거를 적용합니다. include_str! 및 include_bytes! 매크로는 그렇지 않습니다.

[input.tokenization]

토큰화

결과 문자 시퀀스는 이 장의 나머지 부분에 설명된 대로 토큰으로 변환됩니다.

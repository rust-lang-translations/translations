The Rust Reference

[names.preludes]

프렐류드

[names.preludes.intro]

_프렐류드(prelude)_는 크레이트의 모든 모듈 스코프에 자동으로 도입되는 이름들의 집합입니다.

이러한 프렐류드 이름은 모듈 자체의 일부가 아닙니다. 이들은 이름 확인 중에 암시적으로 조회됩니다. 예를 들어, Box와 같은 것이 모든 모듈의 스코프에 있더라도, 현재 모듈의 멤버가 아니기 때문에 self::Box로 참조할 수 없습니다.

[names.preludes.kinds]

여러 가지 다른 프렐류드가 있습니다:

  • 표준 라이브러리 프렐류드
  • 외부 프렐류드
  • 언어 프렐류드
  • macro_use 프렐류드
  • 도구 프렐류드
[names.preludes.std]

표준 라이브러리 프렐류드

[names.preludes.std.intro]

각 크레이트에는 단일 표준 라이브러리 모듈의 이름으로 구성된 표준 라이브러리 프렐류드가 있습니다.

[names.preludes.std.module]

사용되는 모듈은 크레이트의 에디션과 크레이트에 no_std 속성이 적용되었는지 여부에 따라 달라집니다.

에디션no_std 적용되지 않음no_std 적용됨

2018std::prelude::rust_2018core::prelude::rust_2018
2021std::prelude::rust_2021core::prelude::rust_2021

std::prelude::rust_2015 및 std::prelude::rust_2018은 std::prelude::v1과 동일한 내용을 가집니다.

core::prelude::rust_2015 및 core::prelude::rust_2018은 core::prelude::v1과 동일한 내용을 가집니다.

[names.preludes.extern]

외부 프렐류드

[names.preludes.extern.intro]

루트 모듈에서 extern crate로 임포트되거나 컴파일러에 제공된(예: rustc의 --extern 플래그) 외부 크레이트들은 _외부 프렐류드_에 추가됩니다. 만약 extern crate orig_name as new_name과 같이 별칭으로 임포트했다면, 대신 new_name 심볼이 프렐류드에 추가됩니다.

[names.preludes.extern.core]

core 크레이트는 항상 외부 프렐류드에 추가됩니다.

[names.preludes.extern.std]

크레이트 루트에 no_std 속성이 지정되지 않은 한 std 크레이트가 추가됩니다.

[names.preludes.extern.edition2018]

에디션 차이: 2015 에디션에서는 외부 프렐류드에 있는 크레이트들을 use 선언을 통해 참조할 수 없습니다. 따라서 일반적으로 이들을 스코프로 가져오기 위해 extern crate 선언을 포함하는 것이 표준적인 관례입니다.

2018 에디션부터 use 선언은 외부 프렐류드의 크레이트를 참조할 수 있으므로, extern crate를 사용하는 것은 관용적이지(unidiomatic) 않은 것으로 간주됩니다.

참고: Cargo를 사용할 때 alloc 및 test와 같이 rustc와 함께 제공되는 추가 크레이트들은 --extern 플래그로 자동으로 포함되지 않습니다. 이들은 2018 에디션에서도 extern crate 선언을 통해 스코프로 가져와야 합니다.

#![allow(unused)]
fn main() {
extern crate alloc;
use alloc::rc::Rc;
}

Cargo는 오직 절차적 매크로(proc-macro) 크레이트에 대해서만 proc_macro를 외부 프렐류드로 가져옵니다.

[names.preludes.extern.no_std]

no_std 속성

[names.preludes.extern.no_std.intro]

기본적으로 표준 라이브러리는 크레이트 루트 모듈에 자동으로 포함됩니다. std 크레이트가 루트에 추가되며, 암시적인 macro_use 속성과 함께 std에서 내보낸 모든 매크로를 macro_use 프렐류드로 가져옵니다. core와 std 모두 외부 프렐류드에 추가됩니다.

[names.preludes.extern.no_std.allowed-positions]

std 크레이트가 자동으로 스코프에 추가되는 것을 방지하기 위해 크레이트 레벨에서 _no_std 속성_을 적용할 수 있습니다.

이 속성은 다음 세 가지 일을 수행합니다:

[names.preludes.extern.no_std.extern]
  • std가 외부 프렐류드에 추가되는 것을 방지합니다.
[names.preludes.extern.no_std.module]
  • (위에서 설명한 대로) 어떤 모듈이 표준 라이브러리 프렐류드를 구성하는 데 사용될지에 영향을 줍니다.
[names.preludes.extern.no_std.core]
  • std 대신 core 크레이트를 크레이트 루트에 주입하고, core에서 내보낸 모든 매크로를 macro_use 프렐류드로 가져옵니다.

참고: 표준 프렐류드 대신 코어 프렐류드를 사용하는 것은 크레이트가 표준 라이브러리를 지원하지 않는 플랫폼을 대상으로 하거나, 의도적으로 표준 라이브러리의 기능을 사용하지 않으려는 경우에 유용합니다. 이러한 기능들은 주로 동적 메모리 할당(예: Box 및 Vec), 파일 및 네트워크 기능(예: std::fs 및 std::io)입니다.

경고: no_std를 사용한다고 해서 표준 라이브러리가 링크되는 것을 방지하지는 않습니다. 크레이트에 extern crate std;를 넣는 것은 여전히 유효하며, 의존성(dependencies)들이 이를 링크할 수도 있습니다.

[names.preludes.lang]

언어 프렐류드

[names.preludes.lang.intro]

언어 프렐류드에는 언어에 내장된 타입과 속성들의 이름이 포함됩니다. 언어 프렐류드는 항상 스코프에 있습니다.

[names.preludes.lang.entities]

여기에는 다음이 포함됩니다:

  • 타입 네임스페이스
    • 불리언 타입 — bool
    • 텍스트 타입 — char 및 str
    • 정수 타입 — i8, i16, i32, i64, i128, u8, u16, u32, u64, u128
    • 머신 의존적 정수 타입 — usize 및 isize
    • 부동 소수점 타입 — f32 및 f64
  • 매크로 네임스페이스
    • 내장 속성
[names.preludes.macro_use]

macro_use 프렐류드

[names.preludes.macro_use.intro]

macro_use 프렐류드에는 extern crate에 적용된 macro_use 속성을 통해 임포트된 외부 크레이트의 매크로들이 포함됩니다.

[names.preludes.tool]

도구 프렐류드

[names.preludes.tool.intro]

도구 프렐류드에는 타입 네임스페이스에 있는 외부 도구들의 이름이 포함됩니다. 자세한 내용은 도구 속성 섹션을 참조하십시오.

[names.preludes.no_implicit_prelude]

no_implicit_prelude 속성

[names.preludes.no_implicit_prelude.intro]

_no_implicit_prelude 속성_은 크레이트 레벨이나 모듈에 적용되어, 해당 모듈 또는 그 하위 모듈들의 스코프에 표준 라이브러리 프렐류드, 외부 프렐류드, 또는 도구 프렐류드를 자동으로 가져오지 않아야 함을 나타낼 수 있습니다.

[names.preludes.no_implicit_prelude.lang]

이 속성은 언어 프렐류드에는 영향을 주지 않습니다.

[names.preludes.no_implicit_prelude.edition2018]

에디션 차이: 2015 에디션에서 no_implicit_prelude 속성은 macro_use 프렐류드에 영향을 주지 않으며, 표준 라이브러리에서 내보낸 모든 매크로가 여전히 macro_use 프렐류드에 포함됩니다. 2018 에디션부터는 macro_use 프렐류드도 제거합니다.
